Q2: How do I know these predictions are accurate?

Data summary and motivations: The movie rating and genre prediction algorithms were trained on a dataset of 39.2K movie posters found on Kaggle. The movies date back from 1874 to 2022, IMDb ratings are on a scale of one to ten, and span across 26 different genres. The main motivations of this project was to explore transfer learning in computer vision applications while exploring new machine learning tools, data visualization, and web frameworks. In particular, the fast.ai python library for transfer learning, Huggingface spaces to host machine learning applications, the Gradio API to create flexible data flows, RESNET18 a pre-trained model which already learned rich feature representations for a wide range of different object categories, the Bootstrap web framework for a highly iterable responsive web user-experience, and Chart.js for responsive charts built on top of JavaScript. Using these tools we can build a shareable, self-contained dashboard which incorporates an end-to-end deep learning model.

IMDb Rating regression model evaluation: With the IMDb rating regression algorithm, the model was able to achieve a (Mean Absolute Error) MAE of 0.79. This outperforms naive models such as median, mean, and mode, which achieve an MAE of 0.88, 0.89, and 1.02 respectively. When we sample some predicted scores in our test set, we can see that the algorithm has some predictive qualities in assessing movie quality (or IMDb rating). Q3: How can we aggregate and compare images across other dimensions?

Color Spectrum of All Movies from 1874-2022

One method to view movie posters at a macroscopic scale would be to obtain the pixel colors (a red, green, blue number ranging from 0-255) for each poster and show the distribution of colors (% of red pixels, % of black pixels, etc.). This results in a combination of 16.7 million or 256×103 number of colour combinations. In order to efficiently represent 16.7M colors on a 2D plane we can reduce the number of color combinations by taking each pixel's CIELAB color space and find the closest primary parent color, reducing the number of colors to 700. The reason why I used the CIELAB color space to group colors is that unlike the RGB color model, the CIELAB coordinate is designed to approximate human vision. Colors which have CIELAB coordinates that are similar to each-other are similar in color. In contrast, RGB values which are close together, may not For example, a pixel ■ with a blue CIELAB color space cof 4.1×10−6, 2.81×10-7, -4.77×10-6 would aggregate to a blue parent color #006de0 ■ since their CIELAB numerical coordinates are close to each other (small Euclidean distance between the three values) and as a result are visually similar in color. In contrast, the RGB values are (81, 127, 222) and (0, 109, 224) respsectively, so grouping RGB values which have a small Euclidean distance between them would not yield a smooth color gradient. On the other hand, certain shades of red (#ff0300) only appear %0.00004262 of the time. We can now take this data and analyze our color spectrum across several dimensions: time, genre, and ratings.

Color Spectrum of Movies by Decade

Next let's look at the color spectrum across each decade. I excluded movies from 1870 and 2020 since they had less than three movies per decade. We can see four general trends when we look at the color spectrum across time.

1880-1900, Monochrome Black and White (n=57):This was the dawn of the movie industry where directors were experimenting with early motion picture technologies and movie posters were often screen stills. The blue colors we see were from the colorized black and white screen stills from series of short films called Monkeyshines. An interesting piece of trivia is that this film is possibly the "first film to have been produced in the USA, and the first film series."

1910-1950, The Dawn of Color (n=3.2K): Due to the advent of four color wet process for inks, known as CMYK (Cyan, Magenta, Yellow, and Key (black)) gave way to Monkeyshines.
